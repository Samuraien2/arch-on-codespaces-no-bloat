# Contributing to Arch-on-Codespaces

Thank you for your interest in contributing!
We welcome all kinds of contributions—bug reports, feature requests, code improvements, and documentation updates.

---

## How to Contribute

### 1. Fork the Repository

Click “Fork” at the top right of [this repository](https://github.com/syntaxMORG0/Arch-on-codespaces) to create your own copy.

### 2. Clone Your Fork

```sh
git clone https://github.com/yourgithubusername/Arch-on-codespaces.git
cd Arch-on-codespaces
```

### 3. Create a Branch

Create a new branch for your changes:

```sh
git checkout -b your-feature-branch
```

### 4. Make Your Changes

- Make your improvements or bug fixes.
- Please ensure your changes follow a clear and consistent style.

### 5. Test Your Changes

- If you are updating shell scripts or configs, consider testing them in a Codespace.
- Make sure existing functionality is not broken.

### 6. Commit and Push

```sh
git add .
git commit -m "Describe your changes"
git push origin your-feature-branch
```

### 7. Open a Pull Request

Go to your fork on GitHub and click “Compare & pull request”.
- Include a clear description of what you changed and why.

---

## Guidelines

- **Descriptive Commits:** Use clear and concise commit messages.
- **Small Changes:** Break large changes into smaller, focused pull requests when possible.
- **Respect Licensing:** Your contributions will be under GPL v3.

---

## Reporting Issues

- Use [GitHub Issues](../../issues) to report bugs or suggest features.
- Please provide as much detail as possible, including steps to reproduce if relevant.

---

Thank you for making Arch-on-Codespaces better!
